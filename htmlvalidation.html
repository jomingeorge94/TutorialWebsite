HTML - Web Standard and Validation

It is important to follow the Web Standard by the people who develop websites as well as the browsers. Whever you have completed a webpage, make sure that you validate your page with the W3C Markup Validaton Service as it will keep your page up to the standard and help you free off from errors. Web pages which follows the Web Standard are easier for the search engine (Google Chrome, Internet Explorer, etc) to access and read the file and to show the output on the website.

Accessibility

Accessibility is an important part in HTML Standard, as it will make it easier for peopl with disabilities. This will help them to change their website experience much more easier. In the Web Standard, blind people can still use the website long as there something that reads out what is happening on the screen. Also for people that has bad eye-sight can use an option to rearrage the website size.

W3C

Inventor of the web Tim Berners-Lee created W3C which maintains the Web Standards. W3C stands for World Wide Web Consortium, created in October 1994.

Web Validation

W3C Markup Validation Service is the validator that you have to run your website to check each single pages of your site against the Web Standards. Before you run the validator you have to check that you have used the right DOCTYPE because it will give you errors if you haven't met its standards. After scanning your file it will either say that you have passed or a list of errors that it has found which did not met its standards. You can validate your CSS file with W3C as well as your XHTML and XML files.
